# Memory

> 用途：这是整个 skill 的总记忆入口。每次调用时先读本文件，再按这里规定的顺序读取子记忆和资料文件。

## 一、 核心目标

- 本 skill 的目标是帮助单账号完成从选题到成稿的全流程创作。
- 当前版本使用当前 AI 直接生成最终文案，不走多模型评分流程。
- 最终输出统一为：标题、正文、3 个话题。

## 二、 记忆层结构

本 skill 只保留一层总记忆和三类子记忆，避免记忆分散。

### 1. 总记忆

- 当前文件：`memory.md`
- 职责：记录最高优先级的总规则、文件关系、读取顺序、更新边界。

### 2. 账号稳定记忆

- 文件：`01-account-positioning/account-profile.md`
- 职责：记录账号长期稳定信息。
- 典型内容：账号定位、受众、商业模式、表达口吻、禁区、平台适配。

### 3. 账号动态记忆

- 文件：`01-account-positioning/notes.md`
- 职责：记录近期变化、用户反馈、方法论替换、重要提醒。
- 典型内容：用户新偏好、明确否定项、近期有效方法、平台差异提醒。

### 4. 查询方法记忆

- 文件：`03-research/memory.md`
- 职责：记录长期可复用的查询经验和筛选方法。
- 典型内容：高价值搜索方向、长期观察对象、查询避坑点、筛选口径。

## 三、 读取顺序

每次调用 skill，按下面顺序读取：

1. `memory.md`
2. `01-account-positioning/account-profile.md`
3. `01-account-positioning/notes.md`
4. 如果需要做信息查询，再读取 `03-research/memory.md`
5. 再按任务需要读取 `02-asset-library/` 和 `04-copywriting/` 下的资料

## 四、 更新规则

### 允许自动更新

- `01-account-positioning/notes.md`
- `03-research/memory.md`

### 谨慎更新

- `01-account-positioning/account-profile.md`

要求：

- 只有稳定结论、被多次验证的信息，才允许进入账号档案。
- 如果是模型自动补充的长期判断，必须标记状态，例如“待确认”或“自动补充”。

### 当前文件的更新边界

`memory.md` 只允许更新：

- 总体目标变化
- 读取顺序变化
- 文件职责变化
- 极高优先级的总提醒

不要把运行过程中的临时信息写进本文件。

## 五、 高优先级总提醒

- 先读记忆，再读素材，再做查询，再生成文案。
- 平台规则只在生成阶段分流。
- 查询结果只保留长期有价值的结论，不保留无意义的临时堆积。
- 当前版本不使用评分环节，不输出候选稿。
