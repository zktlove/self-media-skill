# 小红书配图提示词框架

这个文件不是具体示例词，而是给 skill 和 MCP 复用的通用骨架。
原则只有一条：固定风格骨架，具体主题全部走变量，不把历史示例写死。

## 一、固定内容（始终保留）

1. 视觉风格
- 可爱手绘涂鸦
- 线条柔和圆润
- 有轻微手工感

2. 色彩方向
- 米白或浅奶油底色
- 以马卡龙柔色为主
- 颜色干净，不脏不灰

3. 版式结构
- 小红书竖版配图
- 3:4 构图
- 主体明确，视觉重心集中
- 多留白，画面不要太满

4. 信息表达
- 可以有少量点题短字
- 不要大段文字
- 重点突出，层级清楚

5. 画面元素
- 可使用圆角矩形、对话气泡、箭头、虚线
- 可搭配 Q 版人物或轻量涂鸦图标
- 整体要干净，不要拼贴截图感

6. 输出约束
- 不要 logo
- 不要水印
- 不要二维码
- 不要与主题无关的主体、道具或场景

## 二、变量部分（每次都替换）

1. 内容主题
- 来自当次文案，不允许沿用历史示例词

2. 主体角色
- 根据主题决定人物、物体或场景主体

3. 主体动作
- 根据主题决定动作，不预设做饭、烘焙、办公等固定动作

4. 点题短字
- 如有需要，只能围绕当前标题和核心卖点生成

5. 三张图的差异化定位
- 第 1 张：封面主视觉
- 第 2 张：信息拆解图
- 第 3 张：氛围场景图

## 三、给 Skill / MCP 的拼装规则

1. 固定框架只提供风格、色彩、构图、约束，不提供具体题材。
2. 变量内容全部从 `copy_text` 中提取，不从示例 prompt 中提取。
3. 如果输入框架文本同时包含“固定内容”和“变量部分”，只读取“固定内容”。
4. 明确禁止把示例中的对象词带入最终 prompt，比如：
- 蛋挞
- 蛋糕
- 甜甜圈
- Q版厨师

## 四、推荐拼装模板

你正在为小红书图文内容生成配图。
请严格遵循以下风格框架：{固定风格框架摘要}。
文案标题：{title}。
文案摘要：{summary}。
核心话题：{topics}。
本张图片定位：{variant_label}。
画面要求：{variant_instruction}。
请输出适合小红书发布的竖版视觉，3:4 构图，主体突出，画面干净。
可以有极少量点题短字，但不要大段文字，不要 logo，不要水印，不要二维码，不要截图拼贴感。
不要沿用示例主题，不要出现与当前文案无关的主体、道具或场景。
